Este site foi desenvolvido para a realização do desafio de Front-end, com o objetivo de realizar o processo seletivo para o estágio na empresa Aiko.

O primeiro passo para o desenvolvimento, foi adicionar o mapa na página por meio de uma biblioteca recomendada. Para facilitar a diferenciação de cada tipo de equipamento, busquei por imagens correspondente a cada um deles e usei para marcar a localização. Ao selecionar um dado equipamento, são fornecidas as principais informações. Além disso, optei por adicionar um botão para que o usuário possa visualizar melhor esses dados, que quando selecionado faz aparecer os dados à direita do mapa, bem como a opção de obter um histórico do estado de funcionamento do equipamento escolhido.

Para cumprir esses objetivos e adicionar as funcionalidades ao site, foi necessário realizar a leitura de arquivos do tipo JSON. Como esse projeto foi o primeiro contato prático que tive com a utilização desse tipo de arquivo, precisei pesquisar como realizar esses processos da melhor forma. Minha primeira escolha foi utilizar a API XMLHttpRequest para acessar os dados do JSON, porém isso funcionou apenas em um primeiro momento. Logo, precisei encontrar outra forma que foi importar esses arquivos e tratá-los como módulos JavaScript.

Para realizar o tratamento dos dados e conseguir apresentar o histórico de estados de cada equipamento avaliei que a maneira mais fácil seria por uma tabela, utilizando um looping que percorre todos os dados do JSON, no qual em cada ciclo é criada uma tabela com duas linhas contendo data e estado. Em seguida, optei por adicionar uma funcionalidade que faz com que a tela desça automaticamente para a tabela, de forma a facilitar a visualização pelo usuário, e também um botão que volta para o topo do página e fecha a tabela. Por fim, decidi por limitar o histórico aos 10 últimos estados e adicionar um botão para o caso de o usuário desejar ver o histórico completo, de forma a otimizar a página e melhorar sua usabilidade.

Como item adicional, quis criar uma seção extra na página que contém um gráfico para mostrar visualmente quantos equipamentos se encontram em cada um dos estados. O objetivo desse gráfico é facilitar o monitoramento pelo usuário e fornecer uma visão geral sobre a frota. Para criá-lo, aprendi a utilizar a API Google Chart, utilizando da documentação oficial.

Dessa forma, desenvolvi o site tentando levar em consideração um design que fosse agradável visualmente e de fácil utilização pelos usuários, levando em consideração as necessidades principais do público-alvo, além de buscar transparecer na interface o princípio de inovação tecnológica fornecido pela empresa Aiko. 

Fazendo esse projeto foi possível adquirir diversas novas noções de desenvolvimento de software, como utilizar a biblioteca Leaflet, arquivos JSON, lógica de programação por trás do funcionamento de cada item, e como amarrar tudo isso com uma boa formatação usando HTML e CSS. Com mais tempo de desenvolvimento, eu melhoraria aspectos como: melhorar a responsividade do site, deixar meu código mais limpo e otimizado e utilizar melhores práticas de programação, que acredito que deixei passar por não ter ainda um conhecimento mais aprofundado. De qualquer forma, achei extremamente proveitoso desenvolver esse projeto e agradeceria pela oportunidade de aprender com o time de vocês a melhorar minhas habilidades cada vez mais, para que um projeto como esse deixe de ser um desafio e possa ser tirado de letra!
